PROJECT CONTEXT
UBUNCARE is a calm, emotionally intelligent mental wellbeing product. The website must feel like a natural extension of the mobile app that has already been designed. The web experience should match the mobile product in tone, visuals, structure, and emotional feel.

CORE GOAL
Design and build the UBUNCARE website so it visually and strategically follows the mobile app experience. The website should feel consistent with the app, not like a separate product.

PRIMARY OBJECTIVES

- Align the website design system with the existing mobile app
- Maintain a calm, premium, emotionally safe wellness aesthetic
- Keep the UX clean, guided, simple, and not overloaded
- Reflect the same product logic as the app: mood check-in first, reflection support, calming tools, gentle guidance, crisis support
- Build a website that is polished enough for launch and trustworthy enough to support app submission
- Ensure the site can host Privacy Policy and Terms pages
- Prioritize clarity, emotional trust, accessibility, and mobile responsiveness

BRAND / EXPERIENCE DIRECTION
The website should feel:

- calm
- warm
- modern
- gentle
- trustworthy
- emotionally safe
- premium but simple
- clean and guided, not busy or corporate

VISUAL DIRECTION
Follow the mobile app styling closely:

- soft green / teal primary palette
- clean white or off-white background areas
- rounded cards
- soft borders
- subtle shadows
- simple icon-led feature cards
- large friendly headings
- generous spacing
- soothing, uncluttered hierarchy

Avoid:

- harsh corporate styling
- too many colours
- overly technical visual language
- cluttered layouts
- feature overload
- dark/heavy aggressive wellness styling

MOBILE APP FEATURES TO MIRROR ON WEBSITE
The website should reflect the same core product journey and main tools shown in the app:

Core experience:

- Mood Check-In
- Reflection / Mood Chat
- Quick Calm
- Guided Breathing
- Mood History
- Insights
- Gratitude
- Wellness Tools
- Crisis Support

Within wellness tools, the app currently includes:

- Guided Breathing
- Weekly Intention
- Self-Care Ideas
- Body Scan
- Sensory Grounding
- Emergency Calm

IMPORTANT PRODUCT DIRECTION
Do not present everything equally.
The website should guide users toward the most important actions first.

The website should emphasize a focused MVP experience:

- Mood Check-In as primary
- Reflection support as a core differentiator
- Quick Calm as fast support
- Guided Breathing as a key wellness tool
- Crisis Support as always available and clearly signposted

Secondary tools like gratitude, insights, history, self-care ideas, weekly intention, body scan, sensory grounding, and emergency calm can be shown more softly as supporting tools or future/expanded tools. Keep the website strategically focused.
