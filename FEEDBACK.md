This is actually pretty strong. Much better than the earlier structure.

My honest review:

What’s working well

The site now feels:

calmer
clearer
more launch-ready
more aligned to the app

You’ve done a good job with:

strong hero
clear “How it works”
focused core tools section
visible crisis support
privacy/trust language
download CTA
legal pages in footer

The overall flow is much more like a real product now.

What I would change before launch

1. “Meet your wellness guides” is too prominent

For this MVP, that section is still a bit too big and too early.

Why:

it makes the app feel more AI-chat heavy
you already said the AI chat part is not the main launch focus
it adds complexity before trust/conversion

Better:

move it lower, after crisis/trust
or shrink it into a smaller supporting section 2. “The Science” section is risky as written

This is the biggest issue now.

You currently say things like:

“3× better at noticing mood patterns”
“78% feel more self-aware after 2 weeks”
“Based on early user testing and self-reported wellbeing data”

If you do not have real documented evidence for those numbers, remove or rewrite them.

That kind of claim can create:

trust issues
app review scrutiny
legal/compliance questions

Safer replacement:

rename to “Designed for simple daily support”
talk about short sessions, privacy, gentle routines, and accessible tools
avoid stats unless you can prove them 3. Header link labels are okay, but “Tools” is slightly generic

You could make this more human.

Instead of:

Tools

Try:

Support Tools
Wellness Tools
How We Support You

Not essential, but nicer.

4. Hero is good, but the subcopy can be even sharper

Current message is solid, but I’d make the value clearer.

Current:
“A gentle daily companion for your mental wellbeing.”

Stronger:
“A gentle daily space to check in, reflect, and find calm.”

That matches your app flow better.

5. “No account needed” and “Everything stays on your device”

Only keep these if fully true
This is very important.

If the app does:

analytics
crash reporting
any cloud sync
external API calls later

then this wording may become misleading.

So:

keep it only if technically true
otherwise soften it

For example:

“Designed with privacy in mind”
“Your personal reflections are kept private”
“Built to feel private and respectful”

7. Footer and section duplication can be reduced

You currently have:

Tools
Wellness
Your Guide
Features-like sections

It is fine, but slightly repetitive.

You can simplify the story to:

How it works
Core tools
Wellness support
Crisis support
FAQ

Cleaner is stronger.
